# operational model

logistical framework for the agreement between a [service provider](service provider.md) and a [user](user.md)

<object type="image/svg+xml" data="../../diagrams/operational model.dot.svg">
    <img alt="operational model Diagram" src="../../diagrams/operational model.dot.png" /> <!-- Fallback for non-SVG browsers -->
</object>

Clause: 3.5.9.1

Note 1 to entry: The logistical framework can set restrictions on how a passenger (3.6.2.1) and/or good is received, transported and/or delivered.

History note: Introduced in ISO/TS 14812:2022

## Specializations of operational model

| Class | Description |
| --- | --- |
| [concurrent](concurrent.md) | [operational model](operational model.md) where [services](service.md) can be provided to multiple [users](user.md) at any one time |
| [dynamic route](dynamic route.md) | <transport service> [concurrent](concurrent.md) operation where transported items can only be received or delivered at stopping points within a pre-defined [service](service.md) corridor |
| [fixed route](fixed route.md) | <transport service> [concurrent](concurrent.md) operation where transported items can only be received or delivered at stopping points contained in a pre-defined sequence |
| [paired on-demand](paired on-demand.md) | <transport service> [concurrent](concurrent.md) operation where the [transport provider](transport provider.md) may choose to divert from its path to service a new request from another [transport users](transport user.md) while servicing an earlier transport user |
| [sequential](sequential.md) | [operational model](operational model.md) where [services](service.md) are provided to a single [user](user.md) at any one time |
